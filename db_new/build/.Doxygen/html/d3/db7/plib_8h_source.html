280/* to understand why this macro exists along with its implicit
281 * pl_arg_global_ptr. originally pl_a returned a pointer pl_arg
282 * to the actual item in the arguments array, this was very bad
283 * as when the argument list reached capacity it would overwrite
284 * the pointers memory location which creates a dangling pointer.
285 * this pointer can still be interacted with without error, the
286 * issue with this is that the pointer that we think is pointing
287 * to an argument in our argument list is actually not existant
288 * which means that from a debug point of view if you created an
289 * argument and the list was re-allocated then now for some
290 * reason regardless of how you trigger that argument it still
291 * comes up as false when using pl_arg_run this is of course
292 * because the pointer is refering to something other then what
293 * it is suppost too. to combat this i changed the code so an
294 * index is returned instead of a pointer, then after all of the
295 * args are set you can use either &PL_ARGS[index] or PL_GETI(i)
296 * to grab a pointer to an argument, now this was a good fix but
297 * now one line has turned into two lines, one for definition
298 * and a second for actually getting the pointer from the index.
299 * the way this is resolved is by using the following macro which
300 * is a copy of the PL_A macro but for the pl_arg_global_ptr func
301 * the pl_arg_global_ptr function will first check to see if the
302 * arg has been created if it has then it will search for it in
303 * the argument list and return a pointer to it if it finds it.
304 * if the arg has not been made then the arg will be offloaded
305 * into the pl_a function and then returned as a pointer. this
306 * means that if you initialize a var using the PL_P function
307 * every time you call the variable it will perform a search in
308 * the argument list for the argument which removes all risk of
309 * dangling pointers as the information is updated per-call and
310 * isent static and overwritable. */
311#define PL_P(...) pl_arg_global_ptr((pl_arg){__VA_ARGS__})
312pl_arg *pl_arg_global_ptr(pl_arg in);
